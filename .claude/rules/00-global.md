# Global rules

## Always true

- treat finance-critical behavior as correctness work, not cosmetic work
- keep one source of truth for each concept
- preserve a clean separation between deterministic logic and AI-generated commentary
- prefer explicit data flow over hidden convenience layers
- do not introduce infrastructure that the local single-user setup does not need

## Planning rule

Before broad edits, say:
- objective
- touched files
- validations to run

## Implementation rule

- make one coherent change set per task
- reuse shared context files instead of rewriting the same project background
- when a new pattern is needed, put it in the relevant context or skill once rather than re-explaining it in multiple files

## Output rule

When finishing work, report:
- what changed
- what was validated
- any remaining risk or deferred follow-up
