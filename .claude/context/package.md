# Context package

This package exists so Claude can load a curated project context bundle from one import.

## Loading guidance

Read only the sections needed for the current task:
- product or roadmap questions -> `project-overview.md`, `phasing-and-ai.md`
- cross-cutting engineering decisions -> `engineering-style.md`, `ai-collaboration-style.md`, `build-and-test.md`
- backend work -> `architecture-and-structure.md`, `backend-guidelines.md`, `backend-style-kotlin.md`, `api-and-contracts.md`, `testing-style.md`
- frontend work -> `frontend-guidelines.md`, `frontend-style-react.md`, `design-system.md`, `api-and-contracts.md`
- database work -> `database-guidelines.md`, `database-style-sql.md`, `domain-invariants.md`, `testing-style.md`
- finance-critical rules -> `domain-invariants.md`, `phasing-and-ai.md`

Read shared style files before inventing a new local pattern.
